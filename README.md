  Incident Reporting System & Dashboard
Project Overview
This project is a real-time incident reporting system that allows users (crowd) to report accidents. The system collects data and provides a dashboard for authorities to monitor and analyze incidents.

Problem Statement
Accidents are often underreported or reported late. This system helps improve response time and decision-making by providing real-time incident data to authorities.

Technologies Used
- Python (Backend)
- Java (Application logic)
- PostgreSQL (Database)

How It Works
1. Users submit incident reports through the system  
2. The backend processes the input data  
3. Data is stored in the PostgreSQL database  
4. Authorities access a dashboard to view and analyze reports  

Features
- Crowd-sourced data collection  
- Real-time reporting  
- Secure dashboard for authorities  
- Data storage and management  

Impact
- Faster emergency response  
- Better decision-making  
- Improved road safety analysis
